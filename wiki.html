ログインボーナス (第一期限定)

1日1回ログインすると自動でアイテムが付与されました。入手できるアイテムは以下の3種で、組み合わせることで経験値の塊を作成できました。

  • けいけんの石
  • けいけんの砂
  • 珍しい水

SCTE (第一期限定)

Super Crafting Table EX（SCTE）は、カスタムアイテム作成に必要な専用作業台です。第二期以降では廃止されています。

  • HUBから行ける「クラフトエリア」に固定設置
  • モバイル版プレイヤーもクラフトエリアで作成可能

競羊 (第二期)

羊のレースに賭け金をかけて配当を得るギャンブルシステムです。単一がけ形式で、1頭の羊に賭けます。

  • 1位 → 2倍の配当
  • 2位 → 1.5倍の配当
  • 3位 → 0.5倍の配当
  • 最低購入額：100マネー
  • 最大購入額：1,000,000マネー

スクリプトイベントコマンド

TPS モニタ (arai:tps)

説明：サーバーのTPS（Ticks Per Second）を5秒間のサンプリング期間で測定し表示します。

エイリアス： arai:tps, arai:tps_check, a:tps, a:tps_check

使用方法：

/scriptevent arai:tps
/scriptevent a:tps

出力：測定されたTPS値を実行プレイヤーに表示します。

TPS ブロードキャスト (arai:tps_share)

説明：最後に測定されたTPS結果をワールド内の全プレイヤーにブロードキャストします。

エイリアス： arai:tps_share, arai:send_tps, a:tps_share, a:send_tps

使用方法：

/scriptevent arai:tps_share
/scriptevent a:tps_share

出力：最新のTPS測定結果を接続中の全プレイヤーに送信します。

Commander API ステータスチェック

説明：capi:worldスコアボードのcurrent_tick値を監視することで、外部Commander APIアドオンがインストール・活動しているかを確認します。

エイリアス： asa3:capi_check, a:capi_check

使用方法：

/scriptevent asa3:capi_check
/scriptevent a:capi_check

出力：

  • ✓ Commander API is Available — APIが稼働中
  • ✗ Commander API is Not Available — APIが検出されていないか稼働していない

ファンクションコマンド

オートインデックス

説明：ワールドのデータインデックスを維持するため、3分ごとに自動実行されます。

実行方式：スクリプトシステムにより3分間隔で自動トリガーされ、手動実行は不要です。

ステータス：ビヘイビアパックのfunctions/ディレクトリに登録されています。カスタム機能を実装する場合は、データパックのfunctionsフォルダに関数を追加してください。

システムフィーチャー

1. 統合スクリプトイベントハンドラー

すべてのスクリプトイベントは、以下の機能を持つ中央集約型ハンドラーシステムを通じて処理されます：

  • スクリプトIDプレフィックスの正規化（arai:, asa3:, または短形式のa:）
  • 適切なハンドラーへのイベントルーティング
  • エラーハンドリングとログ管理

2. Commander API ステータス管理

以下の方法でCommander APIの可用性を監視します：

  • 検出方法：現在のティックスコアボード監視
  • チェック間隔：5分ごとに自動実行
  • 永続状態：他のシステムがアクセスできるようAPI状態をダイナミックプロパティに保存

3. ワールド自動化

  • オートインデックス：3分ごとにfunction indexを実行
  • ワールドクリーンアップ：カスタマイズ可能なスケジュールでワールド状態を管理
  • スペクテイターモード管理：オペレーターがゲームモードを切り替える際のテレポーション処理

4. メインメニューシステム

  • プレイヤー参加配布：新規プレイヤーに自動的にメインメニューアイテムを付与
  • API対応：利用可能な場合はCommander APIを使用して拡張機能を提供
  • フォールバック対応：Commander API未対応の場合はバニラ互換版を提供

5. サーバー転送システム

設定されたBedrock サーバーへのプレイヤー転送に対応：

  • サーバーエイリアスショートカット
  • シームレスな転送実行
  • エラーハンドリングとユーザーフィードバック

6. ワープシステム

G Warp機能を提供：

  • 位置ベースのテレポーテーション
  • サーバーリストの統合
  • プレイヤー国家の検出

オスミウムシステム

オスミウムについて

オスミウムは、ワールドに追加されたカスタムオア素材で、ツールやアイテムのクラフティング用の追加リソースティアを提供します。標準的なマインクラフト素材と比べて、より高い耐久性と効果を備えています。

オスミウムオア

  • ブロックID： asa3:osmium_ore
  • 採掘時間：4秒（鉄以上のツルハシが必要）
  • ドロップ：生のオスミウム
  • 爆発耐性：3.0
  • 入手方法：ワールドに自然生成

オスミウムアイテム

生のオスミウム

  • アイテムID： asa3:raw_osmium
  • 用途：オスミウムオアの採掘で入手
  • 精錬：かまどで精錬してオスミウムインゴットを作成可能

オスミウムインゴット

  • アイテムID： asa3:osmium_ingot
  • 用途：ツールと装備のクラフティング素材
  • スタックサイズ：64

技術情報

依存関係

  • @minecraft/server (v1.13.0以上) — コア サーバーイベントとワールドAPI
  • @minecraft/server-ui (v1.2.0以上) — プレイヤーFormUI（ActionFormData、ModalFormData）
  • @minecraft/server-admin (v1.0.0-beta以上) — サーバー管理機能（プレイヤー転送）
  • Commander API アドオン（オプション） — 拡張API機能のため

バージョン情報

  • パックバージョン：1.0.2
  • 最小ゲームバージョン：1.21.70

インストール方法

  1. [BHP]ASA3フォルダをbehavior_packsディレクトリに配置
  2. オプションで[RES]ASA3リソースパックを追加（ビジュアル強化用）
  3. ワールド設定で両方のパックを有効化
  4. function indexがワールドのデータパックで利用可能か確認

トラブルシューティング

スクリプトイベントが動作しない

  • ワールド設定でアドオンが有効化されているか確認
  • ワールド設定でスクリプトイベントが有効化されているか確認（実験的機能 → スクリプトイベント）
  • 正しいスクリプトID形式を使用しているか確認

Commander API が検出されない

  • Commander API アドオンがインストール・有効化されているか確認
  • APIが稼働中の場合、capi:worldスコアボードが存在する必要があります
  • asa3:capi_checkを実行して手動でステータス確認

オートインデックスが実行されない

  • データパックのfunctionsフォルダにfunction indexが定義されているか確認
  • コンソールでエラーメッセージを確認（利用可能な場合は/script debug ...を使用）

注記

  • すべてのスクリプトイベントコマンドは、フル形式（arai:, asa3:）および短形式（a:）の両方に対応しています
  • このシステムはサーバー環境向けですが、すべてのBedrock ワールドで動作します
  • 最良の結果を得るために、[RES]ASA3リソースパックと組み合わせて使用してください
